# Book Rules

1. The book is about decisions, not gear.
2. No processor is recommended unless it serves a musical purpose.
3. Movement is preferred over unnecessary complexity.
4. Printing audio is treated as a creative act.
5. Every chapter must include a practical exercise or studio mission.
6. Diagrams must clarify workflows, not decorate pages.
7. Markdown is the master source.
8. PDFs are exports, not source files.
9. Avoid filler to reach page counts.
10. The repository is the source of truth, not any single chat conversation.
